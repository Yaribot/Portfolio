    📖 Description

    Simdav VR is a maritime defense simulation project for Naval Group, developed during my apprenticeship at Studio Nyx. The goal of the project is to train new recruits to defend a warship against enemy attacks. The simulation includes one instructor station used to configure the scenario, and two recruit stations where players must cooperate to protect the ship.

    🎯 My role

    - Gameplay Programmer
    - UI Programmer

    ⚡ Challenges

    The main challenges were designing a neutral entity spawning system with movement management, as well as integrating and programming the UI.

    🎯 Challenge: Neutral entity spawning and movement system

    How can we create a system that spawns and despawns neutral entities while managing their movement? These entities simulate maritime traffic lanes and must not be targeted by the player.

    👉 Solution : I implemented an object pooling system for neutral entities to improve performance, avoiding runtime instantiation and destruction. Their movement consists of traveling in a straight line along predefined lanes. They spawn from specific points and return to the pool when reaching an off-screen collider at the end of their path.

    💡 What I would improve : I would consider combining a factory method with the object pooling system to easily support multiple types of neutral entities.

    🎯 Challenge: UI integration and programming

    How can the UI be designed to configure the simulation (number and type of enemy entities, weapon types, etc...)?

    👉 Solution : I created a class responsible for collecting all input field values and configuration settings for each player. These configurations can be saved, creating a mission entry in the mission menu along with a JSON file containing all the data. From there, the mission can either be launched directly or modified, which also updates the JSON file.

    💡What I would improve : I would refactor the system using a command pattern, which better aligns with the single responsibility principle.
